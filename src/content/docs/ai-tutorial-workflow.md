# AI 入门工作流：从问题到可复用流程

很多 AI 使用问题不是模型不够强，而是任务没有被拆清楚。把一次性提问整理成稳定流程，可以让输出更可控，也方便团队复用。

## 一次好的 AI 任务包含什么

一个可复用任务通常包含四层信息：

1. 背景：说明业务、受众、已有材料和目标。
2. 输入：列出模型可以直接使用的原始信息。
3. 约束：明确字数、语气、格式、禁止事项和判断标准。
4. 输出：规定结果结构，最好给出字段或示例。

## 推荐流程

先把需求写成自然语言，再让 AI 帮你补齐缺失信息。确认方向后，把提示词沉淀成模板。

```text
目标：把一段产品介绍改写成小红书风格种草文。
输入：产品卖点、目标用户、价格、使用场景。
约束：不夸大功效，不使用绝对化表述。
输出：标题 5 个，正文 1 篇，评论区引导 3 条。
```

## 什么时候需要迭代

如果输出泛泛而谈，通常是输入缺少场景。如果输出风格不对，通常是没有提供范例。如果输出无法直接使用，通常是验收标准不够清晰。

把这些问题记录下来，下次直接补进模板。
