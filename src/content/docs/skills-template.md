# Skills 文档模板：把能力写成可执行说明

Skills 适合沉淀重复性强、流程固定、需要特定知识或工具的任务。写 Skills 文档时，重点不是描述能力有多强，而是让执行者知道什么时候使用、如何使用、怎样判断完成。

## 推荐结构

```markdown
# Skill 名称

## 何时使用

说明触发条件和不适用场景。

## 输入

列出必须信息和可选信息。

## 流程

把执行步骤拆成清晰动作。

## 验收

说明什么结果算完成。
```

## 写作原则

描述要具体，避免只有抽象目标。比如不要只写“优化文章”，而要写“检查标题、结构、事实、语气和可执行建议”。

## 维护方式

每次真实使用后，把失败案例补进文档。好的 Skill 会随着使用场景变多而越来越清晰。
