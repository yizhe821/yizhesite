# Codex 协作文档：从需求到验收

和 Codex 协作开发时，最有效的方式是给出目标、边界和验收方式。需求可以不完美，但要尽量说明你希望用户看到什么变化。

## 推荐描述方式

```text
目标：新增文档页。
入口：首页顶部 AI 文档、搜索框下方 AI 文档。
布局：左侧目录，右侧正文。
数据：先用静态 Markdown，包含标签、分组、日期。
验收：点击入口可以进入文档页，正文能渲染 Markdown。
```

## 提供上下文

如果项目已有设计风格、路由方式、组件库或数据结构，说明这些信息可以减少来回确认。Codex 会优先沿用现有模式。

## 验收清单

- 页面入口是否可达。
- 目录和正文是否同步。
- Markdown 标题、列表、表格、代码块是否正常显示。
- 静态数据字段是否足够支持后台扩展。

当功能变复杂时，可以先让 Codex 做一版静态闭环，再逐步接入真实接口。
